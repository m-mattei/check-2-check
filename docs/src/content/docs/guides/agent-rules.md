# Agent Rules

These rules are strictly enforced for all AI agents (Antigravity/Opencode) working on the Check-2-Check repository.

1.  **ALWAYS use OpenSpec**: Every change proposal, technical design, and task list must be tracked within the `openspec/` directory.
2.  **ALWAYS document changes and organize properly**: Each feature must have a corresponding entry in this Documentation Center.
3.  **ALWAYS consider refactoring docs at scale**: Do not allow documentation to become fragmented; synthesize ephemeral specs into this central knowledge base.
4.  **ALWAYS add a feature flag**: No feature reaches the user without a toggle in the developer options.
5.  **ALWAYS update spec files after changes**: After archiving a change, ensure the master specifications in `openspec/specs` are current.

## Compliance
Failure to follow these rules will require immediate remediation of the documentation structure.
