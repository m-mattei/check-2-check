## Why

To begin development of the Check-2-Check application, we need to scaffold the initial project structure for the Flutter frontend. This change will establish the core directory layout, basic configuration as defined in `AGENTS.md`, and an entry point to start building the UI and core features.

## What Changes

- Initialize a new Flutter project supporting iOS, Android, and Web platforms.
- Set up the proposed directory structure (`lib/models`, `lib/services`, `lib/screens`, `lib/widgets`, `lib/utils`).
- Provide a clean `main.dart` entry point.

## Capabilities

### New Capabilities
- `project-scaffold`: Initial project setup, directory structure, and basic Flutter app shell.

### Modified Capabilities
- (None)

## Impact

- Creates the foundational codebase for Check-2-Check frontend development.
- Establishes the standard Flutter project structure that all future features will build upon.
